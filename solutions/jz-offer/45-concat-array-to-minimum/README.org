最直观的做法是枚举所有的组合，然后比个大小，选最小的组合。但这样显然很慢，
而且组合出来的数字可能会非常大，可能还要涉及到大数的比较。


+分析一下，如果想让组合最小，那么高位就应该尽可能的小。那就从数组中找一个开头最小的数，+
+但是这个数可能又很多位组成，那么可能要遍历完这么多位，然后判断是否合适。
 当第一个数找出来了，其实就是重复操作了，递归继续去找一个数，直到全部找完。+

 这题其实就是要对数组元素进行排序，得到一个最小排序后拼接成字符串即可。
 难点在于怎么想到排序的方法，即 x + y < y + x ，则 x “小于” y, x 应该在 y 之前。

 同时，还需要去证明：

 1.为什么这种对两个数的排序，对于多个数也成立。

 2.为什么排序后，拼接的数就是最小的。

 至于实现其实蛮简单的，任意排序算法根据 ~x + y < y + x, 则 x 在前面，反之 y 在前面~ 这样的规则排序，
 然后拼接即可。
