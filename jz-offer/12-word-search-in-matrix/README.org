要找到一个单词的路径，随机从矩阵中取一个字符作为起点，从这个字符开始
可以向上下左右四个方向去查找，找到下一个字符后，又可以往上下左右去找。

这种需要多个步骤，每个步骤包含多个选项的问题，可以尝试用回溯法解决。
先朝一个方向进行查找，如果查找失败，则返回上一层，从另一个方向查找。

由于不确定矩阵中哪个字符是起点，就需要遍历一遍，把每个字符作为起点
查找一次，于是需要一个双重循环完成遍历。

每次遍历，都以遍历字符作为起始字符，不断地从四个方向查找，查找结束
条件是：
  - 要查找的字串全部找到了 (这种情况就是找到了，返回 true)
  - 查找越界了，超出矩阵的索引范围
  - 查找的元素被查找过了

最终找到了返回 true， 找不到返回 false 即可
